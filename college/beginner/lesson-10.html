Lesson 10: Exception Handling

Write robust code that handles errors gracefully

Introduction

Exception handling allows your program to handle errors gracefully without crashing. It's a crucial skill for writing reliable code.

Try-Except Block

The basic structure for exception handling:

try:
    # Code that might raise an exception
    result = 10 / 0
except ZeroDivisionError:
    # Handle specific exception
    print("Cannot divide by zero")

Multiple Exceptions

try:
    number = int(input("Enter number: "))
    result = 10 / number
except ValueError:
    print("Invalid input")
except ZeroDivisionError:
    print("Cannot divide by zero")
except Exception:
    print("An error occurred")

Finally Block

Code in finally always executes, whether exception occurred or not:

try:
    file = open("test.txt", "r")
    content = file.read()
except FileNotFoundError:
    print("File not found")
finally:
    file.close()

Raising Exceptions

def check_age(age):
    if age < 0:
        raise ValueError("Age cannot be negative")
    return f"Age is {age}"

try:
    check_age(-5)
except ValueError as e:
    print(f"Error: {e}")

Common Exceptions

  • ValueError: Invalid argument value
  • TypeError: Wrong data type
  • IndexError: Index out of range
  • KeyError: Dictionary key not found
  • FileNotFoundError: File doesn't exist
  • ZeroDivisionError: Division by zero

Best Practices

  • Catch specific exceptions first
  • Use generic Exception as last resort
  • Always use finally for cleanup
  • Provide meaningful error messages
  • Don't silently ignore exceptions
