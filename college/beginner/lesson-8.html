Lesson 8: Dictionaries and Sets

Work with key-value pairs and unique collections

Introduction

Dictionaries store data as key-value pairs, while sets store unique, unordered values. Both are essential for efficient data handling.

Dictionaries

A dictionary is a mutable collection of key-value pairs created using curly braces {}.

my_dict = {"name": "Alice", "age": 25, "city": "NYC"}
student = {1: "Alice", 2: "Bob", 3: "Charlie"}

Accessing Dictionary Values

person = {"name": "Alice", "age": 25}
print(person["name"])      # Output: Alice
print(person.get("age"))   # Output: 25
print(person.get("city", "Unknown"))  # Output: Unknown

Dictionary Methods

person = {"name": "Alice", "age": 25}
person["city"] = "NYC"     # Add new key
del person["age"]          # Delete key
keys = person.keys()       # Get all keys
values = person.values()   # Get all values
items = person.items()     # Get key-value pairs

Sets

A set is an unordered, mutable collection of unique items created using {}.

my_set = {1, 2, 3, 4, 5}
fruits = {"apple", "banana", "cherry"}
empty_set = set()  # Note: {} creates a dict

Set Operations

set_a = {1, 2, 3}
set_b = {3, 4, 5}
print(set_a.union(set_b))        # {1, 2, 3, 4, 5}
print(set_a.intersection(set_b)) # {3}
print(set_a.difference(set_b))   # {1, 2}
print(set_a.issubset(set_b))     # False

Key Points to Remember

  • Dictionaries use key-value pairs
  • Dictionary keys must be unique and immutable
  • Sets store unique values only
  • Sets are useful for removing duplicates
  • Use dictionaries for structured data
  • Use sets for membership testing
